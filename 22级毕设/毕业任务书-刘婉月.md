## 一、 毕业设计（论文）的任务与目标

随着全民健身国家战略的推进，高校内体育场馆使用频次大幅增加。然而，目前多依赖人工登记或基本信息表格的传统场馆管理模式暴露出效率低下、同步滞后等缺点，特别是在高峰期极易引发同一时间段内的场地超员与并发冲突。
本课题主要任务旨在应用基于 Python 的成熟 Web 开发生态，设计并落地一套适配高校真实运营场景的线上体育场馆预约管理系统（MIS），彻底消除传统人工登记的效率瓶颈与时段资源超售痛点。

**主要目标**：
1. 搭建场馆管理的数字化改造闭环，实现师生端自助预约和后台信息化统筹调度。
2. 重点提出并实现场馆防并发冲突管理机制，确保高并发预约请求下的场地时段分配具有唯一准入状态。
3. 提供基础化、有效的数据展示分析，为后勤管理的信息化决策提供保障。

## 二、 毕业设计（论文）的核心工作内容

系统整体采用成熟的 B/S 两级架构，以具备极高开发效率与生态的轻量化 Python Web 框架 Flask 结合 MySQL 关系型数据库搭建，核心工作分解如下：

### 1. 业务底层建模与技术架构开发
*   充分调研现有体育场馆日常运营流，对系统需求建模（系统流程图与角色用例图）。
*   进行 MySQL 数据库精细化设计（必须包含管理用户、场地数据、预约快照记录），建立良好的物理表约束以消除数据冗余，保证高频查询效率。

### 2. 攻防型发并发冲突监测机制设计（重点亮点业务）
*   **持久层防线**：在 MySQL 预约记录表中强行加上联合唯一索引（例如：场地ID+生效日期+起始时间+结束时间）。
*   **接口锁策略**：后端基于 Flask-SQLAlchemy 利用乐观锁（版本号校验）或悲观锁（排他控制方案）保障同批次的竞选操作仅最终一人成功。
*   **高并发验证**：必须单独撰写模拟多用户并发抢票的独立测试脚本来覆盖上述逻辑是否强壮。

### 3. 多角色端应用层系统功能设计
*   **面向师生使用的前台（C端客户端）**：完成系统注册、认证体系，建立实时的预约状态大屏与看板、实现基于精细时间轴的单个场地占用下发操作及历史管理等交互操作。
*   **后台管理控制端调度系统**：实现基础的超管权限划分；场馆及下挂场地资源基础信息的统一管理并支撑人工释放异常预约任务。
*   **轻量数据大屏研发**：整合已入库记录，结合图表开发场地热度排行榜、资源闲置率数据及文字性运营指标面板展示开发。

### 4. 联调、定稿与真实场景验证部署
*   部署至校级体育场馆做小范围试运营测试以暴露流程死角并调优补漏。
*   收集实验反馈，撰写高标准本科学术论文及参与答辩的辅助物料整理。
